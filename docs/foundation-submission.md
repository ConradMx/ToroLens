# Foundation Submission Narrative

ToroLens is positioned as the canonical Toronet web reference application demonstrating production-grade SDK integration, transaction intelligence, identity resolution, server-side architecture patterns, and developer onboarding workflows.

## Ecosystem Value

- Shows developers how to build against Toronet through the official SDK.
- Provides reusable wallet, transaction, TNS, role, and analytics flows.
- Demonstrates server-side validation, error handling, rate limiting, and normalization.
- Documents extension points so future teams can fork the project and add product-specific Toronet capabilities.

## Eligibility Checks

Before submission, confirm:

- Foundation rules allow SDK-wrapper/reference applications.
- Read-only wallet and transaction inspection satisfies ecosystem utility expectations.
- Any direct API fallback, if introduced later, is acceptable when SDK support is unavailable.
- Required proposal artifacts, demo URLs, repo visibility, and roadmap language are aligned with the current Foundation process.

## Current Submission Signals

- SDK-first gateway architecture.
- Normalized data contracts between Toronet and UI.
- Developer reference flows visible in the product.
- Environment documentation and extension guides.
- Test harness for malformed user input.
